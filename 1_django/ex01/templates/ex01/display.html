Display Process of a Static Page

1. Client Request

The user opens a browser and enters a URL, sending an HTTP GET request to the server.

2. Server Receives Request

The web server (e.g., NGINX or Apache) receives the request and passes it to the web application via an ASGI/WSGI interface.

3. Routing

The web framework maps the URL to a specific view function based on routing rules. For example, a URL like /static-page is mapped to a view called static_page_view.

4. View Execution

The matched view function, static_page_view, is executed. Since this is a static page, thereâ€™s no need to fetch data from a database.

5. Template Rendering

The view function calls the template engine to load the HTML template, such as static_page.html, from the templates directory.

If the template has placeholders, the view may supply optional data in a context dictionary.

6. HTTP Response Creation

The rendered HTML content is wrapped in an HTTP response object, which includes necessary headers and a status code (e.g., 200 OK).

7. Response Sending

The response is sent back to the client through the application server, where the browser will receive it.

8. Browser Receives and Renders

The browser receives the HTTP response and renders the HTML content, displaying the static web page to the user.
